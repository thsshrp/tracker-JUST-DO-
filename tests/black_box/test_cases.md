#  Функциональные тесты (Черный ящик)

В данном документе описаны прецеденты (Use Cases), по которым тестировщик проводит ручное функциональное тестирование приложения.

## UC-1: Создание и настройка привычки
* **Предусловие:** Пользователь авторизован.
* **Шаги тестировщика:**
  1. Выбрать опцию "Добавить привычку".
  2. Ввести название (например, "Чтение").
  3. Указать цель (например, "20 минут в день") и категорию.
  4. Нажать "Сохранить".
* **Ожидаемый результат:** Система сохраняет привычку и отображает её в личном списке пользователя.

## UC-2: Отметка выполнения привычки
* **Предусловие:** У пользователя есть минимум 1 активная привычка.
* **Шаги тестировщика:**
  1. Открыть главный экран со списком привычек на сегодня.
  2. Нажать на кнопку "Отметить" (checkbox) рядом с привычкой.
* **Ожидаемый результат:** Элемент визуально меняет состояние на "выполнено", система обновляет streak (серию) пользователя.

## UC-3: Просмотр личной статистики
* **Шаги тестировщика:**
  1. Перейти в раздел "Статистика".
  2. Проверить сводную информацию (выполненные дни, streaks).
  3. Выбрать конкретную привычку для детализации.
* **Ожидаемый результат:** Отображается график выполнения за выбранный период и календарь с отметками.

## UC-4: Поиск и вступление в группу
* **Шаги тестировщика:**
  1. Перейти в раздел "Группы" и ввести в поиск ключевое слово (например, "спорт").
  2. Выбрать группу из результатов поиска.
  3. Нажать кнопку "Вступить".
* **Ожидаемый результат:** Если группа открытая — пользователь сразу становится участником. Если закрытая — отправляется заявка модератору.

## UC-6: Публикация анонимного прогресса в группу
* **Шаги тестировщика:**
  1. Отметить привычку как выполненную (UC-2).
  2. Выбрать появившуюся опцию "Поделиться с группой".
  3. Выбрать группу и нажать "Опубликовать".
* **Ожидаемый результат:** В ленте группы появляется пост о выполнении без указания имени пользователя (анонимно).
