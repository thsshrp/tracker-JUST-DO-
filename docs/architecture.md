# 🏗 Архитектура проекта JUST DO

В этом документе представлены схемы и диаграммы, описывающие архитектуру трекера привычек. Схемы спроектированы предыдущей командой. Исходный код диаграмм представлен в формате PlantUML.

## 1. Физический уровень (Развертывание и Компоненты)

### Диаграмма развертывания
<details>
<summary>Посмотреть исходный код (PlantUML)</summary>

```text
@startuml
node "Клиентское устройство\n(браузер)" {
    component "Веб-интерфейс"
}
node "Веб-сервер" {
    component "Веб-приложение"
}
node "Сервер приложений" {
    component "Сервис пользователей"
    component "Сервис управления объектами"
    component "Сервис аутентификации"
}
node "Сервер базы данных" {
    database "База данных"
}
"Веб-интерфейс" --> "Веб-приложение" : HTTP/HTTPS
"Веб-приложение" --> "Сервис пользователей"
"Веб-приложение" --> "Сервис управления объектами"
"Веб-приложение" --> "Сервис аутентификации"
"Сервисы" --> "База данных" : SQL
@enduml
```
</details>

### Диаграмма компонентов
<details>
<summary>Посмотреть исходный код (PlantUML)</summary>

```text
@startuml
package "Клиентская часть" {
    component "Веб-интерфейс" as WebUI
}
package "Серверная часть" {
    component "Сервис пользователей" as UserService
    component "Сервис управления объектами" as ObjectService
    component "Сервис аутентификации" as AuthService
}
package "Хранилище данных" {
    database "База данных" as DB
}
' Интерфейсы
interface "IПользователи" as IUser
interface "IОбъекты" as IObject
interface "IАутентификация" as IAuth
' Предоставляемые интерфейсы
UserService -- IUser
ObjectService -- IObject
AuthService -- IAuth
' Используемые интерфейсы
WebUI ..> IUser : использует
WebUI ..> IObject : использует
WebUI ..> IAuth : использует
UserService --> DB : доступ к данным
ObjectService --> DB : доступ к данным
AuthService --> DB : доступ к данным
@enduml
```
</details>

## 2. Диаграммы последовательности (Sequence)

### Регистрация и Авторизация
<details>
<summary>Посмотреть код: Регистрация (PlantUML)</summary>

```text
@startuml
actor Пользователь
boundary ФормаРегистрации
control КонтроллерПользователя
entity ПользовательСущность
entity РепозиторийПользователей
Пользователь -> ФормаРегистрации : ввестиДанныеРегистрации()
ФормаРегистрации -> КонтроллерПользователя : отправитьДанные(данные)
КонтроллерПользователя -> КонтроллерПользователя : проверитьКорректностьДанных()
alt Данные корректны
    КонтроллерПользователя -> ПользовательСущность : создатьПользователя(данные)
    КонтроллерПользователя -> РепозиторийПользователей : сохранить(пользователь)
    РепозиторийПользователей --> КонтроллерПользователя : подтверждениеСохранения
    КонтроллерПользователя -> ФормаРегистрации : показатьСообщение("Регистрация успешна")
else Данные некорректны
    КонтроллерПользователя -> ФормаРегистрации : показатьОшибку("Ошибка ввода данных")
end
@enduml
```
</details>

<details>
<summary>Посмотреть код: Авторизация (PlantUML)</summary>

```text
@startuml
actor Пользователь
boundary ФормаАвторизации
control КонтроллерАвторизации
entity ПользовательСущность
entity СервисАутентификации
entity МенеджерСессий
Пользователь -> ФормаАвторизации : ввестиЛогинИПароль()
ФормаАвторизации -> КонтроллерАвторизации : отправитьДанные(логин, пароль)
КонтроллерАвторизации -> СервисАутентификации : проверитьДанные(логин, пароль)
СервисАутентификации -> ПользовательСущность : найтиПользователя(логин)
alt Пользователь найден и пароль верный
    СервисАутентификации --> КонтроллерАвторизации : авторизацияУспешна
    КонтроллерАвторизации -> МенеджерСессий : создатьСессию(пользователь)
    МенеджерСессий --> КонтроллерАвторизации : сессияСоздана
    КонтроллерАвторизации -> ФормаАвторизации : показатьСообщение("Вход выполнен")
else Ошибка авторизации
    СервисАутентификации --> КонтроллерАвторизации : ошибкаАвторизации
    КонтроллерАвторизации -> ФормаАвторизации : показатьОшибку("Неверный логин или пароль")
end
@enduml
```
</details>

### Управление объектами (Привычками)
<details>
<summary>Посмотреть код: Создание объекта (PlantUML)</summary>

```text
@startuml
actor Пользователь
boundary ФормаСозданияОбъекта
control КонтроллерОбъекта
entity ОсновнойОбъект
entity РепозиторийОбъектов
Пользователь -> ФормаСозданияОбъекта : ввестиДанныеОбъекта()
ФормаСозданияОбъекта -> КонтроллерОбъекта : отправитьДанные(данные)
КонтроллерОбъекта -> КонтроллерОбъекта : проверитьДанные()
alt Данные корректны
    КонтроллерОбъекта -> ОсновнойОбъект : создать(данные)
    КонтроллерОбъекта -> ОсновнойОбъект : установитьСтатус("Создан")
    КонтроллерОбъекта -> РепозиторийОбъектов : сохранить(объект)
    РепозиторийОбъектов --> КонтроллерОбъекта : подтверждениеСохранения
    КонтроллерОбъекта -> ФормаСозданияОбъекта : показатьСообщение("Объект успешно создан")
else Данные некорректны
    КонтроллерОбъекта -> ФормаСозданияОбъекта : показатьОшибку("Ошибка ввода данных")
end
@enduml
```
</details>

<details>
<summary>Посмотреть код: Редактирование объекта (PlantUML)</summary>

```text
@startuml
actor Пользователь
boundary ФормаРедактированияОбъекта
control КонтроллерОбъекта
entity ОсновнойОбъект
entity РепозиторийОбъектов
Пользователь -> ФормаРедактированияОбъекта : выбратьОбъект()
ФормаРедактированияОбъекта -> КонтроллерОбъекта : запроситьДанные(идентификатор)
КонтроллерОбъекта -> РепозиторийОбъектов : получитьПоИдентификатору(идентификатор)
РепозиторийОбъектов --> КонтроллерОбъекта : объект
КонтроллерОбъекта -> ФормаРедактированияОбъекта : отобразитьДанные(объект)
Пользователь -> ФормаРедактированияОбъекта : изменитьДанные()
ФормаРедактированияОбъекта -> КонтроллерОбъекта : отправитьИзменения(данные)
КонтроллерОбъекта -> КонтроллерОбъекта : проверитьДанные()
alt Данные корректны
    КонтроллерОбъекта -> ОсновнойОбъект : обновитьДанные(данные)
    КонтроллерОбъекта -> РепозиторийОбъектов : сохранить(объект)
    РепозиторийОбъектов --> КонтроллерОбъекта : подтверждениеСохранения
    КонтроллерОбъекта -> ФормаРедактированияОбъекта : показатьСообщение("Изменения сохранены")
else Данные некорректны
    КонтроллерОбъекта -> ФормаРедактированияОбъекта : показатьОшибку("Ошибка данных")
end
@enduml
```
</details>

## 3. Диаграммы состояний (State)

<details>
<summary>Посмотреть код: Состояния "Основной объект" (PlantUML)</summary>

```text
@startuml
[*] --> Создан : создать()
Создан --> В_обработке : начатьОбработку()
В_обработке --> Завершён : завершить()
В_обработке --> Отменён : отменить()
Завершён --> [*]
Отменён --> [*]
@enduml
```
</details>

<details>
<summary>Посмотреть код: Состояния "Пользователь" (PlantUML)</summary>

```text
@startuml
[*] --> Новый : создатьПользователя()
Новый --> Активный : активировать()
Активный --> Заблокирован : заблокировать()
Заблокирован --> Активный : разблокировать()
Активный --> Удалён : удалить()
Заблокирован --> Удалён : удалить()
Удалён --> [*]
@enduml
```

## 4. Диаграммы активности (Activity)

start
:Ввод регистрационных данных;
:Отправка данных на сервер;

    :Сохранение пользователя;

    stop
